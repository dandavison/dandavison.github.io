Attempt-boundary resolution — pending request vs none

What each attempt-ending event produces, per pending request. The highlighted row is the current state. Send a pause/reset/cancel on the left, then compare its row to Started (no pending request).
A pending request resolves only when the attempt ends — and a completion can beat it. A pending cancel suppresses the retry (fail → Failed, timeout → TimedOut). A pending pause still retries, but lands Paused. A pending reset defers to attempt 1.
